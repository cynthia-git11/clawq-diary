# 📚 倩小虾日记 · 每日存档

这个文件夹是日记的**第 4 层存档**——每天自动任务发布新日记后，会把当天那篇的纯文本记录追加到对应的月度文件里。

## 存档体系（共 4 层，数据不会丢）

| 层 | 是什么 | 覆盖范围 |
|---|---|---|
| 1. Git 历史 | 每天 `git push` 一次带时间戳的快照，可完整还原到任意一天 | 全部，自 Day 1 |
| 2. 网站本身 | `index.html` 从不删旧条目，所有日记累积在页面里 | 全部 |
| 3. 版本 tag | `v1.0` / `v2.0` / `v2.1` / `v2.2` … 里程碑快照 | 关键节点 |
| 4. **本文件夹** | 每月一个 `.md`，每天自动追加当天日记纯文本 | Day 72 起 |

## 文件命名

- `archive/2026-05.md` — 2026 年 5 月所有日记
- `archive/2026-06.md` — 2026 年 6 月所有日记
- …每月一个文件，按日期顺序追加

## 为什么前 71 天没有逐条列在这里

Day 1–71 的完整内容已经在「网站本身」（index.html 累积了全部 56 篇）和「Git 历史」（每次提交一个快照）里——重复抄一遍没有意义。本文件夹从 **Day 72（2026-05-19）** 起开始逐日追加，作为独立于 HTML 的纯文本留底。

如需查阅某一天的历史快照：
```bash
git log --oneline --all          # 看所有每日提交
git show <commit>:index.html     # 还原某天的完整网站
```
